# Отчет по проекту WPF MVVM

## 1. Цель работы

Разработано WPF-приложение для управления списком сотрудников по паттерну MVVM.  
Главное окно показывает список сотрудников в табличном виде, а детальная карточка открывается в отдельном окне и позволяет редактировать все поля записи.

## 2. Архитектура решения

Проект разделен на простые слои:

- `Models` - модель данных `Employee`.
- `ViewModels` - логика представлений `MainWindowViewModel` и `EmployeeDetailsViewModel`.
- `Views` - XAML-окна интерфейса.
- `Services` - работа с JSON-файлами и диалоговыми окнами.
- `Infrastructure` - базовые классы MVVM: уведомления об изменениях, команды и валидация.

Такое разделение нужно для того, чтобы интерфейс, бизнес-логика и работа с файлами не смешивались в одном месте.

## 3. Используемые технологии

- `WPF` - построение настольного интерфейса.
- `MVVM` - разделение UI и логики.
- `ObservableCollection<T>` - обновление списка в интерфейсе.
- `ICollectionView` - поиск и фильтрация без изменения исходной коллекции.
- `ICommand` - обработка действий пользователя через команды.
- `INotifyPropertyChanged` - обновление интерфейса при изменении данных.
- `INotifyDataErrorInfo` - валидация пользовательского ввода.
- `System.Text.Json` - загрузка и сохранение данных в JSON.

## 4. Реализованный функционал

В приложении реализованы следующие возможности:

1. Отображение списка сотрудников с колонками:
   - ФИО
   - должность
   - отдел
   - возраст
   - email
   - телефон
2. Открытие детальной карточки сотрудника двойным щелчком или кнопкой `Редактировать`.
3. Добавление новых сотрудников.
4. Удаление выбранного сотрудника.
5. Поиск по нескольким полям.
6. Фильтрация по отделу.
7. Проверка корректности данных:
   - обязательность полей
   - возраст только числовой и в диапазоне
   - проверка формата email
8. Загрузка данных из внешнего JSON-файла.
9. Сохранение изменений обратно в JSON-файл.
10. Обработка ошибок при чтении и записи файла.

## 5. Демонстрация работы приложения

Сценарий демонстрации:

1. После запуска автоматически загружается файл `Data/employees.json`.
2. В главном окне отображается таблица сотрудников.
3. В поле поиска можно ввести часть ФИО, отдела или должности.
4. В выпадающем списке можно отфильтровать сотрудников по отделу.
5. По кнопке `Добавить` открывается окно создания сотрудника.
6. По кнопке `Редактировать` или двойному щелчку открывается окно редактирования выбранного сотрудника.
7. По кнопке `Сохранить` изменения записываются в текущий JSON-файл.
8. По кнопке `Загрузить` можно выбрать другой внешний JSON-файл.

## 6. Обработка ошибок

Для повышения надежности реализована обработка типовых ошибок:

- файл не найден;
- JSON содержит некорректную структуру;
- ошибка доступа к файлу;
- попытка сохранить некорректные данные;
- ввод нечислового возраста.

Во всех таких случаях пользователю показывается понятное сообщение.

- есть поиск и фильтрация;
